1. 회사 정체성

"유어메이트는 프로젝트 단위로 사업부와 외부 자원을 결합하여 기획부터 실행, 결과물, 정산까지 완성하는 통합 서비스 운영 조직."

전체 흐름

고객 요청 → 분석/견적 → 프로젝트 생성 → 사업부 배정·협업 (1↔N) → 실행 (내부+외부) → 결과물 납품 → 정산/아카이빙

핵심 정책 (사용자 확정)

  • 시스템 = 프로젝트 중심 (사업부 중심 X)
  • 빵빵이 자율 = 일기·분석·제안·일상 쓰기 / 컨펌 필수 = 외부 송신·재무·사용자
  • 대표 권한 = role=admin 자동 인지 (별도 모드 X)
  • 외부 송신 = 초안만, 사람이 검토 후 발송
  • 수의계약 100% (입찰 후순위)
  • 모바일 필수 (직원 절반 이상 외출)
  • 핵심 동기 = 대표 손에서 떼어내기 + 담당자 교체에도 영향 없게

7. 자동 업무표 비전 (Q6 사용자 핵심)

직원의 모든 행위 자동 추적 → 일일/주간 업무표 자동 생성. 사람은 추가 사항만 작성.

10. 두 종류의 *완료* (Q58)

진행중 → 결과물 납품 + 고객 확인 영업 완료 (biz_completed_at) → 외주·외부 인력 정산 재무 완료 (finance_completed_at)
영업 완료 = 매출 인식. 재무 완료 = 영업이익 *확정*. 빵빵이가 영업 완료 후 14일 외주 미지급 알림.
