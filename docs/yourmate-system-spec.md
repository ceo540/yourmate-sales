# 유어메이트 사내 시스템 기획서

> 작성일: 2026-04-05  
> 한 줄 요약: 노션(업무·일정)과 구글시트(매출·원가·수금)로 나눠 관리하던 것을 하나의 웹 시스템으로 통합한 것.

---

## 1. 왜 만들었나

| 이전 | 지금 |
|------|------|
| 노션 프로젝트 DB | 업무·일정 관리 |
| 구글시트 매출보고서 | 매출·원가·수금 관리 |
| → 두 군데 따로 관리, 연결 안 됨 | → 한 화면에서 통합 관리 |
| 노션 월 10만원+ | 자체 시스템 (비용 없음) |
| 회원가입 불가, 권한 제한 | 직원별 권한 분리 |
| AI 연결 어려움 | 빵빵이(AI) 직접 연동 |

---

## 2. 전체 구조

이 시스템의 핵심은 **"한 프로젝트(매출 건)에 모든 정보가 연결"** 되어 있다는 것.  
노션에서 DB끼리 Relation으로 연결했던 것처럼, 여기서도 데이터들이 서로 연결되어 있어요.

### 핵심 데이터 3개 (모든 것의 중심)

| 데이터 | 출처 | 설명 |
|--------|------|------|
| 매출 건 (sales) | 노션 프로젝트 DB + 구글시트 매출보고서 **통합** | 업무·일정과 매출·수금을 한 곳에서 |
| 고객 (customers / persons) | **새로 구축** | 기존엔 발주처 텍스트 한 칸이었던 것 |
| 직원 (profiles) | **새로 구축** | 기존엔 노션 PM 속성(이름 선택)이 전부였던 것 |

이 3개를 중심으로 나머지 모든 데이터가 연결됩니다.

## 3. 메뉴별 설명

### 영업 영역

**리드 파이프라인 (`/leads`)**  
= 기존 구글시트(조민현이 Apps Script로 구축)를 시스템으로 이전한 것

- 아직 계약 안 된 잠재 건들을 단계별로 관리
- 유입 → 회신대기 → 견적발송 → 조율중 → 진행중 → 완료 / 취소
- 담당자 / 서비스 타입 / 소통 경로 / 유입 경로 기록
- 1~3차 소통 이력 기록
- 리마인드 날짜 설정 → D-day 뱃지로 표시
- "매출건으로 전환" 버튼 누르면 → sales로 이동 + 드롭박스 폴더 자동 생성

---

**매출 현황 (`/sales`)**  
= 노션 프로젝트 DB(업무·일정) + 구글시트 매출보고서(매출·원가·수금)를 하나로 합친 것

- 모든 계약 건의 목록
- 담당자 / 발주처 / 매출 / 수금 상태 기록
- 건 하나 클릭 → 프로젝트 허브로 진입 (업무·소통·노트·계약정보 탭)
- 빵빵이(AI)가 여기서 데이터 읽고 쓸 수 있음

---

**사업부별 보기 (`/departments/[사업부]`)**  
= 노션에서 "사업부" 속성으로 필터한 뷰

- 사업부(유어메이트 / SOS / 아트키움 / 학교상점 등)별 건 모아보기
- 각 사업부 안에서 서비스 탭(납품설치 / 교육프로그램 / 교구대여 등)으로 분류
- **교구대여 탭은 렌탈 관리로 바로 이동** (전용 시스템 따로 있음)

---

### 프로젝트 허브 (매출 건 하나 클릭하면 나오는 화면)

노션에서 페이지 하나 열면 본문 + 속성 + 하위 DB가 있는 것처럼,  
매출 건 하나를 클릭하면 탭이 나와요:

| 탭 | 노션으로 비유하면 |
|----|----------------|
| 개요 | 페이지 상단 본문 + AI 요약 |
| 업무 | 하위 태스크 DB (체크리스트 포함) |
| 소통 내역 | 댓글/활동 로그 (→ 개선 예정, 아래 참고) |
| 자유 노트 | 자유 기록 공간 + AI 채팅 |
| 계약 정보 | DB 속성(Properties) 편집 화면 |

**소통 내역 개선 계획:**  
현재는 단순 로그 형태. 노션에 "NEW미팅출장DB"가 있었는데 회의록과 합쳐서 소통 탭으로 통합할 예정.  
기록해야 할 최소 정보:
- 소통 일시
- 누가 소통했는지 (담당자)
- 소통 성격: 전화 / 미팅 / 화상 / 이메일 / 채널톡 / **내부회의** 등
- 소통 내용 메모

> 매출 건뿐 아니라 리드 파이프라인에서도 소통 이력이 같은 구조로 남아야 함.

---

### 드롭박스 연동

드롭박스는 회사의 **중앙 파일 보관소**. 계약서·견적서·사진·기획서·운영계획안 등 프로젝트에 필요한 모든 파일을 여기에 보관.

**이전 방식 (수동 3단계):**
```
매출 건 생길 때마다
1. 노션에 프로젝트 페이지 추가
2. 드롭박스에 폴더 수동 생성
3. 구글시트 매출보고서에 행 추가
→ 매번 3곳을 따로 작업해야 했음
```

**지금 방식 (자동):**
```
시스템에서 매출 건 등록
→ 드롭박스 폴더 자동 생성 + URL 저장
→ 매출 현황에 자동 반영
→ 끝 (노션·구글시트 별도 작업 없음)
```

빵빵이(AI)가 드롭박스 폴더 안의 PDF 파일도 읽을 수 있어서, 기존 폴더에 파일이 있으면 빵빵이에게 요청해 매출 건을 자동 등록하는 것도 가능.

### 특수 관리 (서비스 전용 페이지)

**렌탈 관리 (`/rentals`)**  
= 교구대여 전용 페이지 (일반 프로젝트 구조로는 못 담는 데이터가 있어서 분리)

- 배송일 / 수거일 / 배송·수거 방법 관리
- 품목별 수량 × 기간 × 단가 자동 계산
- 배송·수거 체크리스트
- 등록 시 드롭박스 폴더 자동 생성
- **매출 건(sales)과 연결** → 매출 현황·대시보드에도 자동 반영됨

> 앞으로 다른 서비스도 이렇게 분리할 수 있음.  
> 기준: 일반 프로젝트 구조(업무+계약정보)로 못 담는 데이터가 생길 때.

### 고객 관리

**고객 DB (`/customers`)**  
기존엔 구글시트·노션에 "발주처" 텍스트 한 칸이었던 것을 전용 DB로 구축한 것.  
**기관**과 **담당자**를 분리해서 관리하는 게 핵심.

**기관 (customers)**
- 학교, 기업, 기관 등 발주처 단위
- 이름, 지역, 연락처, 홈페이지, 티어(VIP/일반/신규)
- 매출 건·리드와 연결 → 어느 기관에서 몇 건 계약했는지 파악

**담당자 (persons)**
- 기관 안의 실제 담당자 개인
- 이름, 연락처, 이메일, 직함, 부서
- **소속 이력 추적**: 이직·발령으로 기관이 바뀌어도 담당자 카드는 하나로 유지
- **계약 이력 영구 보존**: 어느 기관에 있든 담당자가 만들어준 계약 건이 계속 쌓임

```
담당자 홍길동
├── 수원교육지원청 (2022~2024) → 당시 계약 건 2개
└── 용인교육지원청 (2024~현재) → 이직 후 계약 건 1개
    → 홍길동이 만들어준 총 계약 3건, 이력 전부 조회 가능
```

> 기관이 주문하기도 하지만, 담당자 한 명이 이직·발령을 통해 거래 기관을 늘려주기 때문에  
> 담당자 데이터가 기관 데이터만큼 중요함.

---

**거래처 DB (`/vendors`)**  
= 협력사·프리랜서 관리 (현재 미완성, 재구축 필요)

- 거래처별 누적 거래 금액
- 계산서 발행 정보 (사업자번호, 발행 이력)
- 어떤 서비스 타입과 연관된 거래처인지
- 원가(납품비·외주비) 지출 시 해당 거래처에 연결

---

**CS 가이드 (`미구현`)**  
= 구글시트로 관리 중인 고객 응대 매뉴얼

- 사업부별·서비스별 응대 스크립트 (전화·채널톡)
- 구성: 전화응대 / 공통사항 / 학교상점-렌탈 / 납품설치 / 아트키움 / SOS / 002Creative 등
- 1차 대응 → 2차 대응 → 3차 문제해결 단계별 안내
- 빵빵이 고객 채널 구현 시 이 가이드를 AI 지식 베이스로 활용 예정

---

### 재무 영역

기존에 구글시트 2개로 쪼개서 쓰던 것을 시스템으로 통합한 것.

```
기존 구글시트 구조
├── 자금일보.xlsx
│   ├── 매출보고서 탭    ← 모든 매출 건 목록 (건명, 계약금액, 원가, 수익률, 수금상태)
│   ├── 매출통계 탭      ← 월별·사업부별 집계
│   ├── 자금현황 탭      ← 계좌별 잔고
│   └── 고정비 탭        ← 인건비·이자·구독료
│
└── 자금 마스터 시트     ← 서비스별 원가계산 구글시트 링크 목록
    ├── 아트키움 원가시트 (외부 구글시트 링크)
    ├── 학교상점 원가시트 (외부 구글시트 링크)
    └── 002Creative 원가시트 (외부 구글시트 링크)
```

**핵심 개념: 건별 원가계산**

매출 건 하나에 원가계산서가 붙어 있음. 구글시트 시절에는 별도 스프레드시트로 관리했던 것을 이제 시스템 안으로 통합.

| 원가 구분 | 항목 | 비고 |
|-----------|------|------|
| 내부원가 | 부가세, 감가상각비, 지급수수료, 관리비 | 계약금액 기준 % 자동 계산 |
| 외부원가 | 프리랜서 비용, 외주 업체비용, 재료비 | 건별 수동 입력 |

→ 원가 합산 → 수익률(%), 순이익 자동 계산

**시스템 내 재무 메뉴 현황**

| 메뉴 | 구글시트 출처 | 구현 상태 |
|------|-------------|-----------|
| 매출 보고서 (`/sales/report`) | 매출보고서 탭 + 매출통계 탭 | ✅ 구현 (원가 집계 포함) |
| 미수금 현황 (`/receivables`) | 매출보고서 탭 필터 | ✅ 구현 |
| 지급 관리 (`/payments`) | 원가계산서 외부원가 | ✅ 구현 |
| 자금현황 (`/cashflow`) | 자금현황 탭 | ✅ 구현 |
| 고정비 관리 (`/fixed-costs`) | 고정비 탭 | ✅ 구현 |
| 급여 (`/payroll`) | 고정비 인건비 | ✅ 구현 |
| 재무 개요 (`/finance`) | 전체 손익 | ✅ 구현 |

> 재무 영역은 관리자(방준영)만 전체 접근 가능.  
> 팀장/팀원은 관리자가 허용한 항목만 볼 수 있음.

> **아직 없는 것**: 결제방법 필드(카드/계좌이체/현금), 서비스 제공일(≠ 계약일)

### 팀 관리 영역

노션에 있던 팀 관련 DB들을 시스템으로 이전하거나 새로 구축한 것.

| 메뉴 | 출처 | 역할 |
|------|------|------|
| 공지사항 | 노션 공지사항 DB 이전 | 전사 공지 + 카테고리 필터 |
| 주간보고 | 노션 주간업무리포트 DB 이전 | 팀원 주간 보고서 작성·제출 **(팀 전체가 매주 빠짐없이 사용 중, 최우선 전환 대상)** |
| 일일업무표 | 노션 일일업무표 DB 이전 | 개인 일일 업무 보고 + 담당자(상사)가 팀원 업무 확인 |
| 근태관리 | 새로 구축 | 엑셀 파일 업로드 → 지각/조퇴 자동 집계 |
| 연차관리 | 새로 구축 | 연차 신청 → 관리자 승인 → 잔여 자동 계산 |
| 경비처리 | 새로 구축 | 개인/법인카드 정산 요청 + 영수증 첨부 |

**주간보고 DB 설계 (노션 구조 기반)**

노션 주간보고는 이름·날짜·담당자 속성 + 본문 테이블로 구성돼 있었음.  
시스템으로 옮길 때 각 칸을 독립 필드로 분리:

| 필드 | 설명 |
|------|------|
| 이번주 목표 | 주초에 세운 목표 |
| 이번주 결과 | 실제 완료된 것 |
| 다음주 목표 | 다음주 계획 |
| 지난주 피드백 | 관리자가 작성 (선택, 필수 아님) |
| 아이디어 | 주간 회의 안건 등 |
| 지원 및 요청 | 도움 필요한 것 |

→ 빵빵이 연동: "이번주 내 업무 정리해줘" → 담당 태스크·매출건 기반 초안 자동 작성

> **일일업무표**: 노션에서는 월별 테이블 형태로 작성. 텍스트 직접 입력 또는 담당 매출 건·업무 페이지 링크 연결하는 방식 혼용.  
> → 시스템: 일일업무표 각 행에 "관련 매출 건" / "관련 업무(태스크)" 연결 필드 추가 예정.  
> 두 기능 모두 현재 데모 상태 → 실DB 전환 필요

**개인 홈 (미구현)**  
노션에서 팀원 각자가 쓰던 개인 대시보드. 현재 시스템에 없음.

```
노션 개인 홈 구조 (팀원마다 커스터마이징 다름)
공통: 일일업무표 / 주간보고 링크
개인마다 다름: 루틴 체크리스트 / 메모장 / 프로젝트 뷰 / Tasks 뷰 등
```
> 시스템에서는 기본 틀만 제공하고 팀원이 커스터마이징할 수 있는 구조로 구현 필요.

### 관리자 영역 (`/admin`)

**직원 관리**  
기존 노션에서는 프로젝트 DB의 "PM" 속성에 이름 선택하는 게 전부였음.  
이제는 로그인 계정과 연결된 실제 직원 데이터로, **이름이 들어가는 시스템 내 모든 곳과 자동 연결됨.**

```
직원 한 명의 데이터가 연결되는 곳
├── 영업: 담당 매출 건, 담당 리드, 담당 렌탈
├── 업무: 담당 태스크, 소통 내역 작성자
└── 팀: 연차 잔여일수, 근태 기록, 경비 신청, 급여 이력
```

직원 카드 하나에서 "이 직원이 담당하는 매출 건, 진행 중인 업무, 연차 현황"을 한 번에 파악 가능.

| 탭 | 내용 |
|----|------|
| 기본 정보 | 연락처 / 비상연락처 / 계좌 / 생년월일 |
| 급여 | 연도별 월급 기록 |
| 원온원 | 1:1 면담 기록 |
| 서류 | 재직증명서 등 서류 관리 |
| 온보딩 | 입사자 체크리스트 |

**조직도**  
= 002코퍼레이션(모회사) → 6개 자회사 트리 구조

### 대시보드 (`/dashboard`) = 개인 홈

노션에서 팀원마다 개인 홈 페이지를 크롬 시작 페이지로 쓰던 것과 같은 개념.  
로그인하면 **본인 기준**으로 필요한 정보가 먼저 나오는 화면.

**모든 직원 공통 (관리자 포함) — 개인 홈 스타일**
- 내 담당 리드 (리마인드 임박한 것)
- 내 담당 매출 건 (진행 중인 것)
- 이번주 내 할 일 (태스크)
- 이번주 주간보고 작성 여부 (미작성이면 바로 작성 버튼)

> 대표·이사도 실무를 직접 치는 구조라서 개인 대시보드가 의미 있음.  
> 전사 재무 현황(매출·미수금·손익)은 별도 재무 대시보드(`/finance`)에서 확인.

**재무 대시보드 (`/finance`) — 관리자 전용**
- 이번달 매출 요약 (사업부별)
- 미수금 현황 (금액 + 건수)
- 월별 손익 추이
- 팀원 주간보고 제출 현황 (이번주 누가 냈고 누가 안 냈는지)

> 노션 개인 홈과 다른 점: 노션은 팀원마다 직접 커스터마이징했지만,  
> 시스템은 기본 틀을 제공하고 나중에 개인화 요소(루틴 체크리스트 등) 추가 예정.

|-----------|--------------|-------------------|-------------|

| 리드 | 전체 | 전체 조회 + 본인 담당 수정 | 본인 담당만 |
| 재무 (매출보고서·미수금·손익) | 전체 | 담당 사업부만 | 접근 불가 |
| 재무 (자금현황·급여·고정비) | 전체 | 접근 불가 | 접근 불가 |
| 팀원 주간보고 | 전체 열람 + 코멘트 | 팀원 것 열람 | 본인 것만 |
| 팀원 근태·연차 | 전체 승인 | 팀원 것 확인 | 본인 것만 |
| 직원 관리 | 전체 | 접근 불가 | 접근 불가 |
| 빵빵이 | 전체 DB | 전체 DB | 본인 담당 건만 |

> 팀장 역할은 지금 당장 쓰지 않아도 시스템에는 이미 준비돼 있음.  
> 직원 카드에서 역할 변경 한 번으로 바로 승격 가능.

## 5. 데이터 연결 구조

노션의 Relation처럼 데이터들이 연결되어 있어요.

```
리드 ─────────────────────→ (전환) → 매출 건
                                         ↑
기관 (customers) ─── Relation ───────────┤
담당자 (persons) ─── Relation ───────────┤  ← 외부: 발주처·거래 상대방
거래처 (vendors) ─── Relation ──── 원가 ─┤
                                         │
직원 (profiles) ──── 담당 PM ────────────┤  ← 내부: 우리 팀원
                                         │
업무 (tasks) ─────── 매출 건에 연결 ──────┤
렌탈 (rentals) ───── 매출 건에 연결 ──────┘
```

> **외부(고객)** = 기관 + 담당자: 우리에게 돈을 주는 쪽  
> **내부(직원)** = profiles: 우리 팀원, 로그인 계정과 연결

## 5. AI 기능 (빵빵이)

빵빵이는 **두 채널**에 동시에 존재함. 둘 다 실제 DB에 읽고 쓸 수 있음.

---

### 채널 1: 시스템 채팅창 (`/api/chat`)
시스템 UI 우하단 채팅 버튼. 로그인한 직원이 사용.

**핵심 기능: 미팅 메모 → 자동 등록**

미팅 내용이나 문의 내용을 그냥 붙여넣으면 → 빵빵이가 구조화해서 카드로 만들어줌.

```
[새 계약건 모드]
사용자: "오늘 ○○학교 미팅. 졸업앨범 촬영 + 제작. 3월 진행. 예산 800만원."
빵빵이: 건명 / 발주처 / 서비스타입 / 금액 정리해서 → "이거 맞아?" 확인 후 등록

[새 리드 모드]
채널톡에서 문의 들어온 내용 붙여넣으면 → 리드 카드 만들어서 등록

[기존 건 업데이트 모드]
"△△학교 건 업데이트 내용이야:" → 기존 건 찾아서 변경사항만 업데이트
```

**데이터 조회·수정 가능한 것**

| 요청 예시 | 빵빵이가 하는 일 |
|-----------|----------------|
| "이번달 매출 얼마야" | 월별 매출 집계 조회 |
| "미수금 있는 거 보여줘" | 미수금 현황 조회 |
| "○○학교 건 완납 처리해줘" | payment_status 변경 |
| "△△ 리드 견적발송으로 바꿔" | 리드 상태 변경 |
| "드롭박스 ○○폴더 파일 목록 줘" | Dropbox 파일 조회 |
| "○○담당자 연락처 찾아줘" | 고객 DB 조회 |

---

### 채널 2: 채널톡 그룹채팅 (`/api/channeltalk`)
채널톡 메신저 안에서 빵빵이 멘션(@빵빵이). 팀 채팅 안에서 업무하면서 바로 호출.

**추가 기능: CS 가이드 내장**  
외부 고객 문의(전화·채팅)가 들어올 때 팀원이 빵빵이한테 물어볼 수 있음.

```
팀원: "SOS 견적 문의 들어왔는데 가격이 얼마야?"
빵빵이: "250만원 (부가세 포함). 거리 멀면 추가비용 가능."

팀원: "렌탈 보증금 언제 환급해줘?"
빵빵이: "검수 완료 후 7영업일 이내."
```

서비스별 응대 기준이 전부 내장됨: SOS / 아트키움 / 학교상점(렌탈·납품·유지보수) / 002Creative / 002ENT

### 빵빵이 발전 방향 (제안)

**단기 — 지금 구조에서 바로 붙일 수 있는 것**

| 아이디어 | 설명 |
|----------|------|
| 리마인드 자동 알림 | 리드에 리마인드 날짜 설정 → 당일 아침 채널톡에 빵빵이가 먼저 알려줌 ("오늘 ○○학교 팔로업 날이야") |
| 주간보고 초안 생성 | 팀원이 "이번주 내 업무 정리해줘" → 빵빵이가 본인 담당 태스크·매출건 기반으로 주간보고 초안 작성 |
| 월말 매출 요약 자동 발송 | 매월 말일 → 빵빵이가 채널톡에 월 매출·수금·미수금 요약 자동 게시 |
| 수익률 분석 질문 | "이번 분기 어느 서비스가 마진 제일 높아?" → sale_costs 기반으로 서비스별 수익률 비교 |

**중기 — 약간의 추가 구현이 필요한 것**

| 아이디어 | 설명 |
|----------|------|
| 드롭박스 PDF 자동 읽기 | 기존 계약서·견적서 PDF 업로드 → 빵빵이가 내용 파악 → 매출건 자동 등록 제안 |
| 미팅 녹음 → 자동 등록 | 미팅 녹음 파일 업로드 → 빵빵이가 내용 정리 → 소통 내역 + 매출건/리드에 자동 반영 |
| 소통 내역 자동 분류 | 팀원이 소통 내용 입력하면 → 빵빵이가 성격 분류(전화/미팅/이메일) + 요약까지 자동 작성 |

**장기 — 고객 채널 빵빵이 (외부용)**

채널톡 고객 채팅창에 빵빵이 연결.  
외부 고객이 문의하면 빵빵이가 1차 응대 → CS 가이드 기반 자동 답변 → 복잡한 건만 팀원에게 넘김.

```
고객 → 채널톡 채팅: "아이패드 10대 2주 렌탈 얼마예요?"
빵빵이: "안녕하세요! 아이패드 대여 문의 주셨군요. 
         기간·수량 확인됐고요, 배송지와 날짜 알려주시면 견적 바로 안내드릴게요."
→ 정보 수집 완료되면 빵빵이가 자동으로 리드 등록
→ 담당자에게 알림
```

> 이게 완성되면 팀원 없이도 초기 문의 응대·정보 수집·리드 등록까지 자동화됨.  
> CS 가이드 시스템 이전이 선행 조건.
